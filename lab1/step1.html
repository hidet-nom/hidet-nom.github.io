WorkSpacesの展開

この章では、AWS上にWorkSpacesを構築します。

1-1-1. AWSアカウントへのログイン

払い出されたIDを使用してAWS環境にアクセスします。

  1. 前の章でAWSマネジメントコンソールにログオンしたままの場合は、一旦ログオフしてください。

  2. Webブラウザでマネジメントコンソールにアクセスします。

  3. 「アカウント」には、アカウントID（12桁の番号）「ユーザ名」「パスワード」を入力し、「サインイン」をクリックします。「ユーザ名」は「IAMユーザ名」です。

    ログインに必要な情報 内容
    AWS アカウントの E メールアドレス アカウント ID（12 桁の番号）258867409727
    ユーザー名 IAMユーザ名 labuserXX(XXは割当番号)
    パスワード Amazon!23

  4. ログイン後、対象リージョンを確認します。画面右上、「サポート」の左隣が「東京」であることを確認します。

  5. 手順4で「東京」以外になっている場合は、プルダウンから「アジアパシフィック (東京) ap-northeast-1」を選択します。

1-1-2. WorkSpacesの展開

既に作成されているVPC（仮想データセンター）の中に、WorkSpacesを構築します。

  1. 左上ペインから「サービス」を選択し、「WorkSpaces」を選択します。

  2. 「WorkSpaces」画面で、左上の青い「WorkSpaces」をクリックします。

  3. 「ディレクトリの選択」画面で**WKSLAB01.lab (d-95672f8654)**を選択し、右下の青い「次のステップ」をクリックします。

  4. 「ユーザーの特定」画面の中段にある「ディレクトリからユーザーを選択」で下の様にlabuserXX(XXは割当番号)と入力後「検索」をクリックし自分に割り当てられた2ユーザーのチェックボックスを選択し、右の青い「選択項目を追加」をクリックします。

    項目 入力内容 備考
    ユーザー名 labuserXX-01 (XXは割当番号) , labuserXX-02(XXは割当番号) XXは01 ~ 15

  5. 下の様に2ユーザー選択されたことを確認し、右の青い「次のステップ」をクリックします。

  6. 「バンドルの選択」画面で下の様にStandard with Windows 10(PCoIP)を選択し 「Windowsバンドルの割り当て」で下の様に言語を全て日本語に選択します。右下の青い「次のステップ」をクリックします。

    バンドル表記はwith Windows 10となっていますが、実際のOSはWindows Server 2016です。実際にWindows 10を使用したい場合は。BYOL環境のご用意が必要です。

  7. 「WorkSpacesの設定」画面で下の様に「Auto Stop」を選択します。右下の青い「次のステップ」をクリックします。 （下の様にエラーが表示された場合でも問題なく進めることができます。）

  8. 「WorkSpacesのレビューと起動」画面で右下の青い「WorkSpacesの起動」をクリックします。トレーニングアカウント権限の影響でリストには何も表示されませんが問題ありません。

  9. 「WorkSpaces」画面で下の様に新しく作成したWorkSpacesがあることを確認します。 下の様に検索フィールドに自分に割り当てられたユーザー名を入力し虫眼鏡アイコンをクリックします。「ステータス」は「PENDING」から作成完了後「AVAILABLE」に変更されます。

    項目 入力内容 備考
    ユーザー名 labuserXX-01 (XXは割当番号) または labuserXX-02(XXは割当番号) XXは01 ~ 15

    新規WorkSpaces作成には、10～20 分程度かかります。
