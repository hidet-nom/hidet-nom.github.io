WorkSpacesの管理

この章では、前節で作成したWorkSpaces(仮想デスクトップ)の設定変更や管理を行います。

1-3-1. WorkSpaces設定の確認（管理者視点）

AWS マネージメントコンソールにサインインします。

  1. Webブラウザでマネジメントコンソールにアクセスします。

  2. 「アカウント」には、アカウントID（12桁の番号）「ユーザ名」「パスワード」を入力し、「サインイン」をクリックします。「ユーザ名」は「IAMユーザ名」です。

    ログインに必要な情報 内容
    AWS アカウントの E メールアドレス アカウント ID（12 桁の番号）258867409727
    ユーザー名 IAMユーザ名 labuserXX(XXは割当番号)　XXは01 ~ 15
    パスワード Amazon!23

  3. ログイン後、対象リージョンを確認します。画面右上、「サポート」の左隣が「東京」であることを確認します。

  4. 手順4で「東京」以外になっている場合は、プルダウンから「アジアパシフィック (東京) ap-northeast-1」を選択します。

  5. 左上ペインから「サービス」を選択し、「WorkSpaces」を選択します。

  6. 左上ペインから「ディレクトリ」を選択後、「WKSLAB01.lab」を選択し「アクション」「詳細の更新」をクリックします。

  7. 「ディレクトリの詳細の更新」画面で、「ユーザーセルフサービスアクセス許可」を展開し「クライアントから WorkSpace を再起動する」「ボリュームサイズを増やす」「コンピューティングタイプを変更する」「実行モードを切り替える」「クライアントから WorkSpace を再構築する」がすべて「有効化」になっていることを確認します。確認後、「キャンセル」をクリックします。 ※「更新」や「更新と終了」はクリックしないでください。

1-3-2. WorkSpaces設定の確認（ユーザー視点）

インストラクターの指示があるまで以下の手順は行わないでください。ここでインストラクターが設定を変更します。

  1. 下の画面で、「再接続」をクリックします。

  2. WorkSpaces(仮想デスクトップ)画面が表示されましたら、WorkSpaces(仮想デスクトップ)画面ウィンドウで、下のようにツールバーの「Amazon WorkSpaces」をクリックします。「WorkSpacesの再起動」が表示されるているのを確認します。

  3. WorkSpaces(仮想デスクトップ)画面ウィンドウで、下のようにツールバーの「自分のWorkSpace」をクリックします。「ディスクサイズの増加」「コンピューティングタイプの変更」「実行モードの切り替え」「WorkSpacesの再構築」が表示されているのを確認します。

  4. 3-1-7 と同様にAWSマネージメントコンソールの「ディレクトリの詳細の更新」画面で、「ユーザーセルフサービスアクセス許可」を展開し「クライアントから WorkSpace を再起動する」「ボリュームサイズを増やす」「コンピューティングタイプを変更する」「実行モードを切り替える」「クライアントから WorkSpace を再構築する」がすべて「無効化」になっていることを確認します。確認後、「キャンセル」をクリックします。 ※「更新」や「更新と終了」はクリックしないでください。

  5. WorkSpaces(仮想デスクトップ)画面をウィンドウ化(全画面表示を中止)して、WorkSpaces(仮想デスクトップ)画面ウィンドウの右上の「✖」をクリックします。

  6. 下のような画面が表示されたら「再接続」をクリックします。

  7. WorkSpaces(仮想デスクトップ)画面が表示されましたら、WorkSpaces(仮想デスクトップ)画面ウィンドウで、下のようにツールバーの「Amazon WorkSpaces」をクリックします。「WorkSpacesの再起動」が表示されなくなったのを確認します。

  8. WorkSpaces(仮想デスクトップ)画面ウィンドウで、下のようにツールバーの「自分のWorkSpace」をクリックします。「ディスクサイズの増加」「コンピューティングタイプの変更」「実行モードの切り替え」「WorkSpacesの再構築」が表示されなくなったのを確認します。

1-3-3. WorkSpacesのログオフ（サインアウト）、切断

  1. WorkSpaces(仮想デスクトップ)画面内で、デスクトップ上のFireFoxを起動します。下のような画面が表示されたら「FireFoxの起動時に必ず確認する」チェックをはずし、「あとで」をクリックします。

  2. FireFoxで好きなWebサイトにアクセスします。（ここでは例として、https://aws.amazon.com/jp/にアクセスしてます。）その後、WorkSpaces(仮想デスクトップ)画面ウィンドウの右上の「✖」をクリックし切断します。

  3. 下のような画面が表示されたら「再接続」をクリックします。

  4. WorkSpaces(仮想デスクトップ)画面内で、先程開いていたFireFoxがそのままであることを確認します。

  5. FireFoxを開いたままサインアウトします。

  6. 下のような画面が表示されたら「再接続」をクリックします。

  7. サインアウト(ログオフ)を行ったのでFireFoxが閉じられています。

1-3-4. クリップボード制御の確認　【オプション】

  1. WorkSpaces(仮想デスクトップ)画面内で、Webブラウザのアドレス等をコピーし、クライアント端末のデスクトップ上のブラウザ等にペースト。逆方向のコピー&ペーストも試します。このWorkSpaces(仮想デスクトップ)はクライアント端末とのクリップボードが許可されているためコピー&ペーストが可能です。

  2. クライアント端末のWindowsメニューを開き「Amazon WorkSpaces」を選択し、クリックします。

  3. 【この手順は省略可能です。】もし、このような画面が表示された場合は、「Skip this version」をクリックします。

  4. WorkSpacesクライアントの右上の歯車アイコンをクリックし、「登録の管理」をクリックします。

  5. 登録コードを入力し、

    項目 入力内容 備考
    登録コード wsnrt+ZJRE7F -

    右下の「登録」をクリックします。

  6. 以下の通り入力し、右下の「サインイン」をクリックします。

    項目 入力内容 備考
    ユーザー名 labuserXX-02(XXは割当番号) XXは01 ~ 15
    パスワード Amazon!23 -

  7. このような画面が表示されたら右下の「はい」をクリックします。

  8. WorkSpaces(仮想デスクトップ)画面が表示されるまでしばらく待ちます。次の手順に進みます。

  9. 新しく開いたWorkSpaces(仮想デスクトップ)画面内で、Webブラウザのアドレス等をコピーし、クライアント端末のデスクトップ上のブラウザ等にペースト。逆方向のコピー&ペーストも試します。このWorkSpaces(仮想デスクトップ)はクライアント端末とのクリップボードが禁止されているためコピー&ペーストができません。
